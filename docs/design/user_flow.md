# スタジオゼブラ 予約管理システム ユーザーフロー

## 1. 一般ユーザーのフロー

### 1.1 新規ユーザー登録フロー
```
トップページ → 新規登録ボタン → 登録フォーム入力 → メール確認 → アカウント有効化 → ログイン完了
```

### 1.2 ログインフロー
```
トップページ → ログインボタン → 認証情報入力 → ダッシュボード表示
```

### 1.3 予約確認フロー
```
ダッシュボード → カレンダー表示 → 日付選択 → 空き時間確認
```

### 1.4 仮予約申し込みフロー
```
カレンダー → 日付・時間選択 → 仮予約フォーム入力 → オプション選択 → 内容確認 → 仮予約申請完了 → 管理者承認待ち通知
```

### 1.5 本予約申し込みフロー
```
カレンダー → 日付・時間選択 → 本予約フォーム入力 → オプション選択 → 内容確認 → 利用規約同意 → 本予約申請完了 → 管理者承認待ち通知
```

### 1.6 予約変更依頼フロー
```
ダッシュボード → マイ予約一覧 → 変更したい予約選択 → 変更内容入力 → 変更リクエスト送信 → 管理者承認待ち通知
```

### 1.7 予約キャンセル依頼フロー
```
ダッシュボード → マイ予約一覧 → キャンセルしたい予約選択 → キャンセル理由入力 → キャンセルリクエスト送信 → 管理者承認待ち通知
```

## 2. 管理者のフロー

### 2.1 管理者ログインフロー
```
管理者ページ → ログイン情報入力 → 管理者ダッシュボード表示
```

### 2.2 予約カレンダー管理フロー
```
管理者ダッシュボード → カレンダー管理 → 日付選択 → 予約状況確認/編集
```

### 2.3 仮予約承認フロー
```
管理者ダッシュボード → 予約承認待ちリスト → 仮予約詳細確認 → 承認/拒否決定 → メール通知 → ステータス更新
```

### 2.4 本予約承認フロー
```
管理者ダッシュボード → 予約承認待ちリスト → 本予約詳細確認 → 承認/拒否決定 → メール通知 → ステータス更新 → Google Calendar更新
```

### 2.5 予約変更承認フロー
```
管理者ダッシュボード → 変更リクエスト一覧 → 変更内容確認 → 承認/拒否決定 → メール通知 → 予約情報更新
```

### 2.6 予約キャンセル承認フロー
```
管理者ダッシュボード → キャンセルリクエスト一覧 → 詳細確認 → 承認/拒否決定 → キャンセル料金確認 → メール通知 → 予約削除
```

## 3. 例外フロー

### 3.1 仮予約期限切れフロー
```
システム → 仮予約期限（1週間前）チェック → ユーザーへリマインダー通知 → レスポンスなし → 仮予約自動キャンセル
```

### 3.2 二重予約防止フロー
```
予約申し込み → 日時重複チェック → 重複あり → ユーザーに通知 → 別日時選択案内
```

### 3.3 予約条件不適合フロー
```
予約申し込み → 利用条件チェック（最低2時間など） → 条件不適合 → ユーザーに通知 → 修正案内
```

## 4. 決済フロー（将来拡張）

### 4.1 事前決済フロー
```
予約確定 → 決済情報入力 → 支払い処理 → 決済完了通知
```

### 4.2 キャンセル料金決済フロー
```
キャンセル承認 → キャンセル料金計算 → 支払い情報通知 → 決済処理
```

## 5. 通知フロー

### 5.1 予約関連通知
- 予約申し込み完了通知
- 予約承認/拒否通知
- 仮予約期限リマインダー（予約日1週間前）
- 予約日前日リマインダー

### 5.2 管理者通知
- 新規予約通知
- 予約変更リクエスト通知
- キャンセルリクエスト通知
- 仮予約期限切れ通知
